function runall_sr_control (in subid, in bidsdir, in sdir, in eegpreproc, in audpreproc)
 

Detailed Description

Performs the same analysis as "runall.m" but considers yet another way of defining audio "without CamEQ" for HI listeners. In this case the audio is extracted in the exact same way for NH and HI listeners rather than inverse filtering the already CamEQ equalized files.

Parameters
subidsubject id (an integer between 1 and 44)
bidsdirthe root of the BIDs directory
sdirdirectory in which the features and model results are stored
eegpreproca flag (0/1) that indicates whether or not to build eeg features. by default, it is assumed that "runall" has already been called and that there thus is no need to build these features again
audpreproca flag (0/1) that indicates whether or not to build audio features. this can by handy with limited signal processing licenses
